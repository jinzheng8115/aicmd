# AICmd P0 UX Improvements Design

中文：本文定义两项 P0 UX 改进：`aicmd doctor` 和“执行结果写入 session”。目标是减少安装/配置排障成本，并让同一个 session 的下一轮对话能引用上一轮命令执行结果。

English: This document defines two P0 UX improvements: `aicmd doctor` and command execution results persisted into sessions. The goal is to reduce setup/debugging friction and let later turns in the same session refer to prior command execution results.

## Goals / 目标

中文：
- 用户能用一个命令检查 AICmd 的安装、模型配置、MCP/search 配置和 shell integration 状态。
- 用户执行命令后，下一轮在同一个 session 中可以引用“刚才的执行结果”。
- 输出保持适合终端阅读，不暴露 API key 或敏感配置内容。

English:
- Users can run one command to inspect installation, model config, MCP/search config, and shell integration state.
- After command execution, the next turn in the same session can refer to the previous execution result.
- Output remains terminal-friendly and does not expose API keys or sensitive config values.

## Non-Goals / 非目标

中文：
- 本期不实现完整 `aicmd config` 菜单。
- 本期不实现 `aicmd update`。
- 本期不做复杂 TUI，也不引入新依赖。
- 本期不把完整 stdout/stderr 无限量写入 session。

English:
- This phase does not implement the full `aicmd config` menu.
- This phase does not implement `aicmd update`.
- This phase does not add a complex TUI or new dependency.
- This phase does not persist unlimited stdout/stderr into sessions.

## Feature 1: `aicmd doctor`

中文：新增内置子命令 `aicmd doctor`，用于诊断本机 AICmd 运行状态。该命令应在加载配置后运行，尽量复用现有路径解析逻辑。

English: Add a built-in `aicmd doctor` subcommand to diagnose local AICmd runtime state. It should run after config loading and reuse existing path resolution behavior where possible.

Checks / 检查项：
- `aicmd` binary path from current executable.
- AICmd version from package metadata.
- Runtime config path and whether `config.yaml` exists.
- Current model id if config can load it.
- MCP config path and whether `mcp.json` exists.
- Whether MCP `search` command is configured.
- Whether the install bin directory appears in `PATH`.
- Whether shell integration appears installed in common shell rc/profile files.

Output / 输出：

```text
AICmd doctor

Binary: ok /home/user/.local/bin/aicmd
Version: ok 0.30.1
Config: ok /home/user/.aicmd/config.yaml
Model: ok deepseek:deepseek-chat
MCP config: ok /home/user/.aicmd/mcp.json
Search: ok configured
PATH: ok /home/user/.local/bin
Shell integration: warning not detected

Suggestions:
- Run: eval "$(aicmd shell-init)"
```

Failure behavior / 失败行为：
- Missing optional items should print `warning`, not crash the process.
- Missing required runtime config should print a direct next step: `aicmd init --from-env`.
- Invalid `mcp.json` should print the parse error context, but not expose secrets.

## Feature 2: Persist Execution Results Into Session

中文：当前 session 会保存用户请求和 AI 生成的命令，但执行后的真实结果不稳定进入 session。本期在用户选择 `execute` 后，把命令执行摘要追加到同一个 session，使后续问题能引用上一轮结果。

English: Today sessions persist the user request and generated command, but actual execution output is not reliably available in later turns. This phase appends an execution summary to the same session after the user chooses `execute`.

Persisted content / 写入内容：
- Generated command.
- Exit code.
- Truncated stdout.
- Truncated stderr.
- LLM command summary when available.

Format / 格式：

```text
Command execution result:
Command:
<command>

Exit code: <code>

STDOUT:
<truncated stdout>

STDERR:
<truncated stderr>

AI summary:
<summary text>
```

Truncation / 截断：
- Keep the stored execution record bounded.
- Default max combined output should be conservative, around 8,000 to 12,000 characters.
- If truncated, append `[truncated]` / `[已截断]`.

Data flow / 数据流：
- `shell_execute` already captures stdout/stderr and calls `summarize_command_output`.
- Change summary generation so it returns the summary text in addition to printing it.
- Add a session helper that appends an assistant-side execution record after command execution.
- Persist the session immediately after appending the execution record.

Safety / 安全：
- Do not persist output when `--dry-run` is active.
- Do not persist output if no session is active.
- Do not write unbounded command output into session files.
- Keep raw command output available in terminal exactly as today.

## Testing / 测试

Manual checks / 手动检查：
- `aicmd doctor` with valid config.
- `aicmd doctor` with missing `~/.aicmd/mcp.json`.
- `aicmd doctor` with missing shell integration.
- Run `aicmd -s p0test 当前目录有多少文件`, choose execute, then inspect the session YAML and verify command result is stored.
- Run a failing command through `aicmd err -- ...` and confirm exit code/stdout/stderr summary can be persisted.

Automated checks / 自动检查：
- `cargo check`
- Existing test suite if available.
- Focused unit tests for output truncation helper if the helper is pure and easy to isolate.

## Product Rationale / 产品理由

中文：这两项改进分别解决“装好了但不知道哪里错”和“同一个 session 不能自然引用上一轮结果”两个高频断点。它们不改变 AICmd 的核心定位，也不引入新概念。

English: These two improvements address two common breakpoints: not knowing what is wrong after installation, and not being able to naturally refer to previous execution results inside one session. They preserve AICmd's focused product shape and avoid adding new concepts.
