# AICmd P0 UX Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build `aicmd doctor` and persist bounded command execution results into the active session.

**Architecture:** Add a focused `doctor_cmd` module for read-only diagnostics, then wire it into `main.rs` as a built-in shortcut. Extend the existing session persistence path with a small helper that appends execution records after shell command execution, without changing raw terminal output.

**Tech Stack:** Rust, clap, serde YAML/JSON, existing AICmd config/session APIs, cargo checks/tests.

---

### Task 1: Add `aicmd doctor`

**Files:**
- Create: `src/doctor_cmd.rs`
- Modify: `src/main.rs`

- [x] **Step 1: Create diagnostics module**

Implement `run_doctor_command() -> Result<i32>` in `src/doctor_cmd.rs`.

The module should:
- Print binary path using `std::env::current_exe`.
- Print version using `env!("CARGO_PKG_VERSION")`.
- Print config path with `Config::config_file()`.
- Print current model id by reading the `model` field from `config.yaml` when it exists.
- Print MCP path using `AICMD_MCP_CONFIG_FILE` or `Config::config_dir().join("mcp.json")`.
- Parse MCP JSON and report whether `mcp.commands.search` exists.
- Check whether the binary directory appears in `PATH`.
- Check common rc/profile files for `aicmd shell-init`.
- Print suggestions for warnings/errors.

- [x] **Step 2: Wire shortcut**

Add `mod doctor_cmd;` to `src/main.rs` and handle `doctor` in `run_pre_config_shortcut` so it works even when `config.yaml` is missing.

Expected behavior:

```bash
aicmd doctor
```

prints diagnostics and exits with code `0`, unless an unexpected IO/runtime error occurs.

- [x] **Step 3: Verify**

Run:

```bash
~/.cargo/bin/cargo check
target/debug/aicmd doctor
```

Expected: cargo check passes, doctor prints terminal-friendly diagnostics.

### Task 2: Persist Execution Result Records

**Files:**
- Modify: `src/config/session.rs`
- Modify: `src/config/mod.rs`
- Modify: `src/main.rs`

- [x] **Step 1: Add session append method**

Add a method on `Session`:

```rust
pub fn add_assistant_note(&mut self, note: String)
```

It should push an assistant `MessageContent::Text(note)`, mark the session dirty, and update tokens.

- [x] **Step 2: Add config persistence helper**

Add a method on `Config`:

```rust
pub fn append_session_note(&mut self, note: String) -> Result<()>
```

It should do nothing when no session is active or dry-run is active. Otherwise, append the note to the active session and persist the session YAML immediately.

- [x] **Step 3: Return summary text**

Change `summarize_command_output` in `src/main.rs` to return `Result<Option<String>>`.

Behavior:
- If stdout/stderr are empty, return `Ok(None)`.
- Continue printing the `AI summary:` section.
- Return the generated summary text when available.

- [x] **Step 4: Store bounded execution record**

Add helpers in `src/main.rs`:

```rust
fn truncate_for_session(value: &str, max_chars: usize) -> String
fn build_execution_session_note(command: &str, code: i32, stdout: &str, stderr: &str, summary: Option<&str>) -> String
```

Use conservative limits so session files do not grow unbounded.

- [x] **Step 5: Wire after execute**

After `run_shell_command_capture` and summary generation, call:

```rust
config.write().append_session_note(note)?;
```

This should happen before `process::exit(code)`.

- [x] **Step 6: Verify**

Run:

```bash
~/.cargo/bin/cargo check
target/debug/aicmd -s p0test-ux "输出 hello"
```

Expected: choose `execute` in the interactive prompt. The session file contains the command execution result record with command, exit code, stdout/stderr, and summary when available.
